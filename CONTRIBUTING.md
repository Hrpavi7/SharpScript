# Contributing to SharpScript

Thanks for your interest in contributing to **SharpScript**!  
This project is currently maintained by a single developer, but suggestions, bug reports, and improvements are welcome.

---

## Scope of Contributions

You can contribute by:

- Reporting bugs
- Suggesting features or improvements
- Improving documentation
- Submitting code changes

Please keep contributions focused and aligned with the projectâ€™s goals.

---

## Reporting Bugs

Before opening an issue:

1. Make sure the issue is reproducible
2. Check if it has already been reported

When reporting a bug, include:
- A clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Your environment (OS, compiler, version)

---

## Feature Requests

Feature requests are welcome, but please note:

- HrSearch aims to stay **lightweight and simple**
- Large or scope-changing features may be declined
- Suggestions should include a clear use case

---

## Code Contributions

### Requirements
- Code must compile without errors or warnings
- Follow existing code style and structure
- Keep changes focused and minimal
- Avoid introducing unnecessary dependencies

### Coding Guidelines
- Use clear, descriptive names
- Prefer simple, readable C code
- Keep functions small and focused
- Comment *why* something is done, not *what* is done

---

## Building the Project

Make sure the project builds before submitting changes:

```bash
mingw32-make
```
